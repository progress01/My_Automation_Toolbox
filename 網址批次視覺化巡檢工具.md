# 📄 網址批次視覺化巡檢工具 (Sitemap Visual Auditor)

本工具為一款自動化的網址巡檢與視覺化報表生成程式。透過內建的 Selenium 驅動無頭瀏覽器 (Headless Chrome)，能批次讀取網址清單、自動擷取網頁畫面，並將結果整合為易於閱讀的單頁面 HTML 儀表板與 PDF 報表。具備中斷續傳機制，特別適合用於大規模網站重構、Sitemap 檢查或上線前的例行性網頁狀態稽核。

## 1. 需準備的輸入檔案格式

請準備一個純文字檔案（**`.txt` 格式**），將需要檢測的網址逐行貼入。

* **內容規範**：每一行必須是一個完整的網址，且必須以 `http://` 或 `https://` 開頭。
* **編碼支援**：程式內建防呆機制，同時支援 `UTF-8` 與台灣 Windows 常用的 `Big5` (ANSI) 編碼，無論從 Excel 匯出或記事本建立皆可直接讀取。
* **資料範例**：
```text
https://www.google.com.tw
https://example.com/about-us
http://test-site.com/api/v1

```


*(註：空白行或非 http 開頭的文字將被程式自動忽略)*

## 2. 預期的產出結果

程式執行完畢後，將會在腳本所在的同一層目錄下生成以下四個項目：

1. **`Sitemap_Audit_progress.db` (SQLite 資料庫)**：
紀錄每個網址的掃描狀態（成功、載入逾時、連線錯誤等）與時間。此檔案為**核心進度檔**，若掃描中斷，下次執行時可自動跳過已完成的網址。
2. **`evidence_shots/` (截圖資料夾)**：
存放所有成功載入網頁的 PNG 截圖檔案。檔名會自動進行安全轉換（過濾掉特殊符號）。
3. **`Report_Sitemap_Audit.html` (互動式 HTML 報表)**：
一個單頁面的視覺化儀表板。內含網址超連結、狀態標籤（成功/失敗，並以顏色區分）以及對應的縮圖。點擊縮圖可放大查看。
4. **`Report_Sitemap_Audit.pdf` (靜態 PDF 報表)**：
當您在主選單執行「[3] 自動輸出 PDF 報表」後，程式會在背景透過 Chrome 引擎將上述 HTML 報表無損轉換為 PDF，方便您作為結案報告或夾帶於 Email 中寄送。

## 3. 注意事項

在使用本工具前，請務必確認您的環境符合以下配置：

* **必備套件安裝**：
首次執行前，請開啟命令提示字元 (Terminal) 並安裝必要的第三方套件：
```bash
pip install selenium webdriver-manager

```


* **瀏覽器相依性**：
您的電腦**必須安裝 Google Chrome 瀏覽器**。程式會自動透過 `webdriver-manager` 下載對應版本的驅動程式，不需手動配置 ChromeDriver。
* **記憶體管理機制**：
為避免瀏覽器長時間執行導致記憶體溢出 (OOM)，程式預設每處理 **50** 個網址就會自動重啟一次背景瀏覽器。這屬於正常優化流程，請勿中斷程式。
* **PDF 圖片路徑限制**：
由於 PDF 轉換引擎的安全限制，報表產生時會使用電腦的「絕對路徑」來讀取圖片。請**先在該資料夾內完成 PDF 轉檔**，若您在轉檔前將 `evidence_shots` 資料夾移動到其他地方，PDF 將會無法正常顯示截圖。
* **重置掃描進度**：
若您修改了目標網站內容並希望「重新截圖」，請在執行選項 [1] 時，於終端機提示「是否要清除舊紀錄並重新掃描」時輸入 `Y`，否則程式會預設讀取資料庫的舊紀錄並跳過掃描。
