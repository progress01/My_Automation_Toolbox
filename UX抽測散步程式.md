# 📄 UX抽測散步程式 (Website Walker)

本工具為基於 `DrissionPage` 與 `Tkinter` 開發的半自動化網頁巡檢與壓力測試系統。程式會模擬使用者的隨機點擊行為（猴子測試），沿著 Sitemap 或指定清單深入內頁進行版面掃蕩，並允許檢測人員隨時透過熱鍵介入標註異常，最終自動匯出完整的巡視軌跡報表。

## 1. 需準備的輸入資料 (Input Data)

本程式**不需要**匯入任何外部的 CSV 或 Excel 檔案，所有測項與目標皆透過啟動後的圖形化介面 (GUI) 直接輸入與設定：

* **📍 Sitemap 起點 (URL Entry)**：
* **格式**：必須是完整的絕對路徑（包含 `https://`）。
* **範例**：`[https://www.reca.gov.tw/sitemap](https://www.reca.gov.tw/sitemap)`
* **說明**：程式會爬取此頁面上的所有乾淨連結，自動過濾掉社群分享、檔案下載等無效目標，並呈現在下方的「測試清單」中供您勾選。


* **📑 抽測內頁數 (Dropdown)**：
* **格式**：數值下拉選單 (0, 1, 2, 3, 5)。
* **說明**：設定每個主單元要額外往下隨機鑽取多少個內頁進行測試。


* **🎯 精準打擊 (特定異常網址)**：
* **格式**：純文字多行輸入，每行一個完整的 HTTPS 網址。
* **說明**：如果您有已經確定的問題網址，可以直接貼在此文字框內，程式會強制將這些網址加入巡邏清單中。



## 2. 預期的產出結果 (Expected Output)

按下「中斷」或等待巡視清單全數跑完後，程式會在同一個資料夾目錄下，自動產生並**主動開啟**一份 Excel 戰報。

* **檔案命名格式**：`UX使用性抽測報告_YYYYMMDD_HHMMSS.xlsx`
* **報表欄位結構**：
| 欄位名稱 | 說明 |
| --- | --- |
| **層級** | 標示該頁面是「主單元」、「內頁」還是手動輸入的「精準打擊」。 |
| **名稱** | 抓取到的網頁節點名稱（例如：便民服務、環保新聞）。 |
| **稽核判定** | 顯示最終狀態，包含「✅ 正常」或您透過鍵盤標註的「⚠️ 排版/RWD異常」、「⚠️ 彈窗遮罩卡死」等。 |
| **網址** | 該次檢測停留的最終 URL。 |
| **頁面字數** | 粗略計算 `<body>` 內的文字量，可用於判斷頁面是否為空或渲染失敗。 |
| **猴子動線軌跡** | **極具價值的黑盒子紀錄**！會列出程式在該頁面點擊過的元件名稱與深度（例如：`[頂部 (0px)] 展開選單 ➡️ [深度 (560px)] 點擊圖片`）。 |



## 3. 執行環境與注意事項 (Caveats & Requirements)

### 環境依賴 (Dependencies)

此工具捨棄了 Selenium，改用更輕量且強大的 `DrissionPage` 來控制瀏覽器，並內建了 GUI 元件。請確保在命令提示字元執行以下安裝指令：

```bash
pip install pandas beautifulsoup4 DrissionPage openpyxl

```

*(註：`tkinter` 為 Python 內建標準庫，無需另外安裝。`openpyxl` 為匯出 Excel 的必備底層套件。)*

### ⚠️ 執行注意事項

* **鍵盤判定機制 (焦點要求)**：程式設計了熱鍵（`Space` 暫停、`1` 排版異常、`2` 遮罩卡死等）。**請注意，這些熱鍵是透過 JavaScript 注入到網頁中的**，因此使用熱鍵時，您的滑鼠焦點必須停留在「網頁視窗」上，若焦點在 GUI 控制面板上按鍵將無效。
* **破壞性操作防護網**：程式內部已寫死了強大的 Regex 與關鍵字黑名單（如：儲存、刪除、submit、登出），以防止猴子測試時誤觸系統按鈕。但若在「具備寫入權限的正式環境後台」執行，仍有極微小機率觸發未防護到的非標準命名按鈕，**強烈建議僅在測試機或公開前台執行此工具**。
* **瀏覽器綁定**：`DrissionPage` 預設會尋找本機安裝的 Chrome 或 Edge 瀏覽器進行接管。執行時會自動跳出一個由程式控制的瀏覽器視窗，請勿手動關閉該視窗，否則會導致程式崩潰。
* **自動拉回機制**：若網站具備嚴格的 Router 重新導向機制（例如未登入強制跳轉首頁），工具在連續被跳轉 3 次後，會自動判定為「⚠️ 異常強制跳轉」並放棄該頁面，這是正常的保護機制，避免陷入死迴圈。
