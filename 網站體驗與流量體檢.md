# 📊 網站體驗與流量體檢

這是一套專為政府機關與大型網站設計的 GA4 自動化稽核腳本。系統採用「三檔鼎立」**架構，能夠一次性載入主流量、補充維度與真實去重人數，並在本地端自動清洗與交叉運算。程式的核心亮點在於**「排除虛榮指標與爬蟲雜訊」，並透過演算法將生硬的數據轉化為具備行動指引（Actionable）的 **11 張戰略報表**。

## 📂 1.輸入資料準備 (三檔鼎立架構)

請確認腳本所在的同層目錄下，具備以下三個由 GA4 匯出的 `.csv` 檔案。
*(若檔案缺失，程式不會當機，對應的報表會自動標示為「缺少檔案」)*

### 1. `GA4_主檔案.csv` (核心動線與象限運算)

* **負責產出**：每日趨勢、動線效率、四象限修繕矩陣、爬蟲防禦名單。
* **必須包含以下欄位**：
`日期 + 小時 (YYYYMMDDHH)`, `到達網頁`, `網頁標題與畫面名稱`, `工作階段來源/媒介`, `事件名稱`, `活躍使用者`, `瀏覽`, `事件計數`, `參與度`, `平均單次工作階段參與時間`。

### 2. `GA4_補充檔案.csv` (裝置與搜尋意圖運算)

* **負責產出**：載具體驗落差表、民眾迷航搜尋分析。
* **必須包含以下欄位**：
`到達網頁`, `裝置類別`, `搜尋字詞`, `活躍使用者`, `事件計數`, `參與度`。

### 3. `GA4_真實人數表.csv` (最高指導 KPI)

* **負責產出**：直接提供給長官/客戶的「真實去重複觸及人數」。
* **必須包含以下欄位**：
`網頁標題與畫面類別` (或 `網頁標題與畫面名稱`), `活躍使用者`。

## 🚀 2.預期的產出結果 (11 大戰略報表)

執行完畢後，系統會產出一份名為 **`化學署_網站體驗與流量深度體檢_三檔綜合總表.xlsx`** 的大型 Excel 報表，並同時將各分頁獨立匯出為 `.csv` 檔。這 11 張表的用途如下：

### 📈 營運與長官 KPI 視角

* **`0_去重複真實KPI總表`**：已排除重複計算的網頁真實觸及總人數。
* **`1_每日結構與破案表`**：呈現每日流量的 Top 3 來源與網頁，方便快速抓出某日流量暴增的「破案線索」。
* **`2_雙軌趨勢繪圖表`**：自動利用正規表示式，將流量切分為「B2B 業務剛需系統」與「B2C 民生科普文章」，方便檢視雙軌運營成效。

### 🛠️ 企劃與 UX 結構視角

* **`3_單元與UX事件矩陣`**：評估各大單元（如：知識地圖、持久性有機污染物）的整體參與度，並**自動計算「檔案下載轉換率」**。
* **`4_大門動線體檢表`**：從「到達網頁 (Landing Page)」分析使用者入站後的後續事件數與平假日維持率。
* **`5_來源歸因排行榜`**：評估各頁面的流量波動屬性，自動打上「爆紅型」或「剛需型」標籤。

### 🚨 工程與資安修繕視角

* **`6_四象限修繕派工單`**：透過演算法將網頁標示為四種狀態（如：`🔥 高流量足跡 ✕ 嚴重卡關`），工程團隊可直接依此清單排程修復。
* **`7_頁面卡關停留`**：列出停留秒數大於 180 秒但參與度低於 40% 的高阻力頁面。
* **`8_載具體驗落差`**：比對同一個網頁在不同裝置（Mobile, Desktop, Tablet）上的參與度，找出 RWD 跑版的嫌疑頁面。
* **`9_民眾迷航搜尋`**：列出站內搜尋的關鍵字。搜尋越多次代表該資訊的動線藏得太深，需調整架構。
* **`10_異常爬蟲防禦名單`**：揪出單人瀏覽量超過 50 頁的「垂直狂刷爬蟲」，供資安團隊阻擋異常 IP，避免伺服器資源浪費。

## ⚠️3. 安裝與注意事項

1. **環境安裝**：請確保 Python 環境已安裝處理引擎，輸入指令：`pip install pandas numpy openpyxl`
2. **避免檔案鎖定**：執行腳本時，**請務必關閉所有相關的 Excel 與 CSV 檔案**，否則會出現 `Permission denied` 的寫入錯誤。
3. **編碼保護**：所有輸出的 `.csv` 皆已採用 `utf-8-sig` 編碼，直接雙擊用 Excel 開啟也不會有中文亂碼問題。
