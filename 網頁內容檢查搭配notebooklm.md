# 📄 網頁內容檢查搭配notebooklm

本工具為基於 `DrissionPage` 與 `Tkinter` 開發的綜合型自動化爬蟲系統。核心目的是精準爬取目標網站的 DOM 結構，濾除無用雜訊，產出純淨的「語意文本 (供 NotebookLM 分析)」以及自動化盤點的「網站三層結構地圖 (Excel)」。系統具備智慧判斷機制，遇到純前端渲染 (SPA) 空殼時，會自動無縫切換至視覺化無頭瀏覽器進行渲染抓取。

## 1. 需準備的輸入資料 (Input Data)

本程式採用圖形化介面 (GUI)，分為四大核心功能分頁，所需的輸入格式各有不同：

| 執行模式 | 輸入要求與檔案格式 |
| --- | --- |
| **1. 單站結構快篩** | **無須檔案**。直接在介面輸入完整網址（例：`[https://www.moenv.gov.tw/](https://www.moenv.gov.tw/)`）。 |
| **2. 清單匯入** | **需準備 `.txt` 文字檔**。每一行代表一個目標網址。若使用 Tab 分隔，系統會自動讀取最後一個欄位作為 URL。 |
| **3. JS動態渲染** | **無須檔案**。直接在介面輸入完整網址。適合已知為高動態、需等待大量 API 沉澱或需手動登入的系統網站。 |
| **4. 多站批次排程** | **需準備 `.xlsx`, `.csv` 或 `.txt` 檔案**。<br>

<br>表格必須嚴格遵守：<br>

<br>・**第一欄 (A欄)**：網站名稱 (例：化學物質管理會報)<br>

<br>・**第二欄 (B欄)**：完整網址 (URL)<br>

<br>*(註：系統會自動略過空行，無須擔心表頭干擾)* |

## 2. 預期的產出結果 (Expected Output)

程式執行完畢後，會在您指定的「檔案儲存位置」中建立一個獨立資料夾（命名格式：`AI萃取_[網站名稱]_[時間戳記]`）。該資料夾內將包含以下核心產出物：

1. **📄 NotebookLM_AI知識庫 (`.txt`)**
* **用途**：專門餵給 LLM 進行 RAG (檢索增強生成) 的極淨文本。
* **內容**：系統已自動剔除 `script`, `style` 等雜訊，並強制展開隱藏的 `<img alt="..">` 與 `title` 提示。文本內會以明確的分隔線切分「導覽/頁首」、「主要內容」與「頁尾/宣告資訊」。


2. **📊 網站三層結構地圖 (`.xlsx`)**
* **用途**：網站架構盤點與稽核。
* **內容**：自動列出 L1 (第一層)、L2 (第二層)、L3 (第三層) 的節點關聯，並附帶該頁面的「頁面標題」、「實際網址」、「內文字數」以及「包含的檔案/圖片數量」。


3. **⚠️ 爬蟲連線異常清單 (`.xlsx`)** *(僅在發生異常時產出)*
* **用途**：除錯與死結盤點。
* **內容**：記錄爬取過程中遇到的 404 死連結、Soft 404 (假正常頁面)、以及連線逾時的網址。


4. **📝 spa_pending_list.txt** *(僅在批次排程模式下產出)*
* 若在大量批次掃描時遇到「純 SPA 網站」，系統為防拖慢整體排程會將其跳過，並記錄在此名單中，建議後續使用「第3分頁：JS動態渲染」單獨處理。



## 3. 執行環境與注意事項 (Caveats & Requirements)

### 環境依賴 (Dependencies)

請開啟命令提示字元 (cmd)，執行以下指令安裝必備套件：

```bash
pip install pandas beautifulsoup4 DrissionPage openpyxl fake-useragent lxml

```

*(註：`fake-useragent` 為選用，若無安裝系統會自動降級使用內建的靜態 UA 池。`lxml` 可大幅加快 HTML 解析速度。)*

### ⚠️ 執行注意事項與防護機制

1. **免安裝版 Chrome 的完美支援 (隨身碟模式)**：
* 程式碼內建了智慧路徑偵測：若您將免安裝版的 Chrome 放在程式同目錄下的 `ChromePortable/chrome.exe`，爬蟲就會優先調用它。這意味著您可以**把整個專案資料夾放進隨身碟，在任何一台 Windows 雙擊執行，完全不會動到系統環境變數**。


2. **WAF 與人機協作機制 (SOS 救援)**：
* 工具並非死板的自動化腳本。若在爬取過程中觸發了 Cloudflare 驗證、Captcha 或是遇到需要登入的頁面，系統會自動暫停，並在瀏覽器上方注入一個**紅色警告橫幅**。您可以安心地手動點擊驗證或登入，完成後點擊該紅色區塊，部隊就會自動恢復推進。


3. **SQLite 本機快取 (`temp_cache.db`)**：
* 程式在執行期間會於同資料夾產生 `temp_cache.db`、`temp_cache.db-wal` 等檔案。這是為了避免對同一個網址重複發送請求而建立的本機離線快取庫，具備優異的記憶體管理。程式結束時會自動清理這些檔案，**若中途當機看到這些檔案殘留，請勿驚慌，可安全手動刪除**。


4. **嚴格的防迷航設計**：
* 預設情況下，程式會鎖定「起始目錄」。例如起點為 `[https://gov.tw/news/](https://gov.tw/news/)`，則爬蟲只會深入 `/news/` 轄下的子網頁，若抓到 `/about/` 的連結則會判定為跨區而停止延伸，確保結構地圖的銳利度。
