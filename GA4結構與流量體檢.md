# 📄 GA4結構與流量體檢

## 1. 需準備的輸入檔案格式

執行本腳本前，請確認同一層目錄下備妥以下三個由 GA4 (Google Analytics 4) 匯出的 CSV 檔案。**請務必確保檔案名稱完全一致，且包含以下指定欄位（欄位名稱需完全符合）**：

### 📂 A. 主檔案：`GA4_主檔案.csv`

這是運算「網頁深度」與「任務效率指標」的核心來源。

* **必備欄位要求**：
* `日期 + 小時 (YYYYMMDDHH)`：用於解析時間與判斷平假日。
* `到達網頁`：用於計算網頁層級深度與聚合來源。
* `網頁標題與畫面名稱`：用於辨識網頁並過濾隱私權、錯誤畫面等雜訊。
* `工作階段來源/媒介`：用於追蹤流量管道。
* `事件名稱`：判斷互動有效性的輔助欄位。
* `活躍使用者`、`瀏覽`、`事件計數`、`參與度`。
* `平均單次工作階段參與時間`：程式內會自動將此欄位重新命名並加權計算為「總參與秒數」。



### 📂 B. 真實人數表（KPI 基準）：`GA4_真實人數表.csv`

這是為了校正 GA4 匯出時常見的加總重複問題，由系統直接去重匯出的基準表。

* **必備欄位要求**：
* `網頁標題與畫面類別` 或 `網頁標題與畫面名稱`（程式內建防呆，兩者皆可自動辨識為「網頁標題」）。
* `活躍使用者`：將作為最終報告的「去重複真實觸及人數」最高指導 KPI。



### 📂 C. 補充檔案（備用）：`GA4_補充檔案.csv`

*(註：目前此版本的程式碼僅在設定區宣告了此檔案，尚未使用於下方的聚合邏輯中，但若您後續要擴充裝置或搜尋意圖分析，需準備此檔)*

* **預計欄位要求**：
* `到達網頁`、`裝置類別`、`搜尋字詞`、`活躍使用者`、`事件計數`、`參與度`。



> 💡 **防呆機制說明**：檔案請盡量使用 `UTF-8` 或是 `UTF-8 with BOM (utf-8-sig)` 編碼匯出。若主檔案的某些數值欄位（如參與度、瀏覽數）為空，程式會自動補 `0` 防止報錯；若缺乏日期欄位，則會自動填入 `1970-01-01` 作為防爆預設值。

## 2. 預期的產出結果

程式順利執行完畢後，將會在腳本所在的同一層目錄下，自動封裝並產出一份完整的 Excel 總表：

* **檔案名稱**：`政府網站_結構與流量深度體檢_結構化總表.xlsx`
* **報表內容**（內含 4 個工作表）：
1. **`0_去重複真實觸及人數_...`**：直接對齊 GA4 真實去重數據的 KPI 基準表。
2. **`1_動線結構與效率指標_...`**：包含各網頁的「加權平均停留秒數」與「任務效率指標」，快速評估動線順暢度。
3. **`2_系統狀態評估矩陣_...`**：將所有網頁自動分群為 4 個象限（如：高流量高阻力、運作常態等），可直接作為工程與企劃團隊的優先待辦清單。
4. **`3_異常流量檢測名單`**：抓出疑似爬蟲或無效過客的異常觸發紀錄，供資安或行銷團隊參考。



## 3. 安裝套件與注意事項

在使用本腳本前，請務必確認以下環境配置與操作限制：

* **必備第三方套件**：
本腳本依賴 `pandas` 進行強大的資料聚合運算，並強制指定使用 `openpyxl` 引擎來寫入多工作表的 Excel 檔案。請開啟命令提示字元 (Terminal) 輸入以下指令進行安裝：
```bash
pip install pandas numpy openpyxl

```


* **檔案佔用問題 (Permission Error)**：
程式執行並匯出報表時，**請務必確保您沒有在 Excel 中開啟 `政府網站_結構與流量深度體檢_結構化總表.xlsx**`。若檔案處於開啟狀態，系統會因為檔案被鎖定而導致寫入失敗並報錯。
* **資料量與記憶體消耗**：
此腳本會在記憶體中合併並運算多個 DataFrame。一般的 GA4 匯出檔絕對游刃有餘，但若您的原始 CSV 檔案極度龐大（例如單檔超過 1GB 或數百萬筆級別），建議在記憶體規格較高的電腦上執行。
* **無效資料自動防呆**：
若 GA4 匯出的主檔案中缺乏任何事件觸發，導致系統無法產出矩陣或爬蟲名單，程式不會當機，而是會貼心地在對應的工作表填入 `['狀態': '資料量不足以運算矩陣']` 等提示字樣。
