# 📄 網站稽核報告批次產生工具 (Batch Word Report Generator)

本工具為一款輕量級的本地端自動化腳本，能讀取 CSV 格式的網站清單，並將特定資料（如網站名稱與前台網址）自動填入 Word 公版範本的指定位置。特別適合用於大規模的網站架構檢核或產生大量制式化稽核報告。這套工具能有效避開企業內部封閉系統的限制，在本地端快速消化上百筆資料，大幅省去人工複製貼上的時間與出錯率。

## 1. 需準備的輸入檔案格式

在執行程式前，請確保腳本同層目錄下有以下兩個檔案：

### A. `data.csv` (資料表檔案)

請提供 UTF-8 編碼的 CSV 檔案，且必須包含以下兩個標題欄位：

* **`檔案名稱`**：作為產出 Word 檔的檔名。系統具備自動解析功能，若檔名包含底線（例：`001_公害糾紛處理系統`），程式會自動提取 `_` 後面的文字作為「網站名稱」。
* **`前台網址`**：該網站對應的完整 URL（例：`[https://sedr.moenv.gov.tw/zh-tw/](https://sedr.moenv.gov.tw/zh-tw/)`）。

### B. `template.docx` (Word 公版範本)

這是您的制式報告樣板。

* **佔位符號設定**：請在文件內（無論是一般段落或表格內部皆可）輸入特定的觸發字串：**`網站名稱/網址`**。
* 程式掃描到該字串後，會自動將其替換為 `網站名稱/網址：[解析出的名稱]/[URL]`，並且**完美保留您公版原有的字體大小與排版樣式**。

## 2. 預期的產出結果

程式執行完畢後，將會產生以下結果：

1. **自動建立 `Output_Reports/` 資料夾**：若原本沒有此資料夾，系統會自動新建。
2. **批次生成獨立報告**：依據 CSV 內的資料筆數，在輸出資料夾內產生對應數量的 `.docx` 檔案。
* *範例：* 成功產出 `001_公害糾紛處理系統.docx`。


3. **無損操作**：原始的 `template.docx` 範本檔案將被完整保留，不會被覆蓋或修改。
4. **防呆範例生成**：若首次執行時系統找不到 `data.csv`，程式會極度貼心地自動建立一份包含兩筆測試資料（環境部相關網站）的範例檔，方便您直接確認格式與測試運作。

## 3. 注意事項

在使用本工具前，請務必確認以下環境配置與邏輯限制：

* **必備第三方套件**：
本腳本依賴 `python-docx` 套件來解析與編輯 Word 文件。若尚未安裝，請開啟命令提示字元輸入以下指令：
```bash
pip install python-docx

```


* **替換邏輯的唯一性**：
目前的文字替換邏輯嚴格鎖定尋找 `網站名稱/網址` 這個字串。若您的公版中有其他欄位（例如：檢測日期、負責工程師等）也需要動態替換，需要回到原始碼的 `replace_meta_info` 函式中新增對應的 `if` 判斷條件。
* **CSV 編碼問題**：
程式預設使用 `utf-8-sig` 讀取 CSV 檔，這能完美相容並解決 Windows 系統下 Excel 另存 CSV 時常見的 BOM（位元組順序記號）與中文亂碼問題。建議所有清單皆使用 Excel 另存為「CSV (UTF-8)」格式。
