# 📄 CSV 網頁查核審查閱卷版本

本工具是一個基於 Python 與 `tkinter` 開發的輕量級轉換器。其核心功能是將含有網址與查核規範的「靜態 CSV 清單」，一鍵轉換為「全動態、支援全鍵盤操作的沉浸式 HTML 查核計分板」。稽核人員可在生成的網頁中，享受左側網頁預覽、右側計分與巨集快速填寫的雙視窗工作流，並具備離線自動存檔功能。

## 1. 需準備的輸入資料 (Input Data)

本程式依賴特定格式的 CSV 檔案。請確保匯入的 `.csv` 檔案（建議使用 UTF-8-BOM 編碼）具備以下確切的欄位名稱，系統才能正確解析與群組化：

| 欄位名稱 | 資料用途與解析邏輯 |
| --- | --- |
| **系統名稱** | 用於對案件進行「群組化（Grouping）」。切換系統時會以此欄位為單位。 |
| **提報問題** | 顯示於右側面板，且系統會**使用正則表達式自動從此欄位萃取出 `http/https` 網址**，並投射至左側預覽區。 |
| **查核教學** | 顯示於右側面板，提供給審查人員的綠色提示文字。 |
| **項次** | 顯示於題目開頭的編號（如：No.1）。 |
| **缺失類別** | 顯示於題目名稱的方括號內（如：【基本要件】）。 |
| **審查重點(涵攝要件)** | 顯示於缺失類別後方，作為該查核項目的主要標題。 |

## 2. 預期的產出結果 (Expected Output)

執行 Python 程式並選擇 CSV 檔案後，系統會在該 CSV 所在的同一個資料夾下，自動產生並開啟一份名為 **`CSV網頁查核計分板.html`** 的網頁檔案。

**HTML 儀表板核心功能：**

* **雙視窗審查**：左側透過 `iframe` 即時預覽受測網址；右側提供評分按鈕與備註輸入框。
* **自動存檔 (Auto-Save)**：所有審查進度與自訂巨集皆會即時儲存於瀏覽器的 `LocalStorage` 中。即使不慎關閉網頁，下次開啟時系統也會提示是否還原進度。
* **最終匯出**：點擊「強制匯出 CSV」或完成所有題目後，系統會生成一份包含「實測判定結果」與「審查備註說明」的最新 CSV 總表，供後續建檔使用。

## 3. 鍵盤快捷鍵與巨集操作 (Hotkeys & Macros)

本系統專為「無滑鼠、高效率」的工作流打造，請確保**鍵盤焦點處於右側計分板**（右上角狀態燈號顯示「🟢 鍵盤就緒」）以使用以下快捷鍵：

| 快捷鍵 | 對應動作 |
| --- | --- |
| **1** | 標記為：✅ 實測屬實 (應給分)，並自動跳至下一題。 |
| **2** | 標記為：❌ 實測不符 (不給分)，並自動跳至下一題。 |
| **3** | 標記為：➖ 略過，並自動跳至下一題。 |
| **W / S** | 往上 / 往下切換題目焦點。 |
| **A / D** | 往左 / 往右切換下一個「系統名稱」群組。 |
| **F2** | 快速將輸入游標定位至當前題目的「備註輸入框 (Textarea)」。 |
| **英文字母 (自訂)** | 觸發設定好的巨集 (Macro) 字串，瞬間填入備註並可設定是否自動給分與跳下一題。 |

## 4. 執行環境與注意事項 (Caveats)

### 環境依賴

本程式僅使用 Python 內建標準庫（`os`, `re`, `csv`, `json`, `time`, `tkinter`, `webbrowser`），**無需安裝任何外部 pip 套件**，具備極高的可攜性。

### ⚠️ 注意事項

* **Iframe 拒絕連線 (X-Frame-Options)**：部分政府網站或高資安網站設有 `X-Frame-Options: DENY` 或 `SAMEORIGIN` 防護，這會導致左側預覽區呈現空白或拒絕連線。此為正常的瀏覽器安全機制，遇到此狀況請點擊畫面上方的「**↗️ 獨立開新分頁測試**」按鈕進行人工檢視。
* **跨瀏覽器進度不互通**：系統的「自動存檔」與「自訂巨集」是綁定在單一瀏覽器的 `LocalStorage` 上的。如果您今天用 Chrome 開啟 HTML，明天改用 Edge 開啟，進度將無法互通。若需轉移環境，請善用畫面上方的「💾 儲存進度」與「📂 讀取進度」功能，將進度匯出為 `.json` 檔案帶著走。
* **網址萃取邏輯**：左側的畫面完全取決於「提報問題」欄位中是否包含完整的 `http://` 或 `https://` 網址。若該欄位沒有網址，左側預覽區將自動隱藏並顯示「此提報項目無附帶網址」的提示。
