# 📄 分頁偵測與提取工具

本工具為一款進階的自動化分頁探測與網址提取腳本，基於 `DrissionPage` 框架開發。系統採用「戰術分離」設計，將「廣域掃描找尋目標」與「封包監聽破解邏輯」拆分為獨立模組。透過正則表達式 (Regex) 與 XPath 多重特徵比對，能自動找出潛在的分頁網址；並結合半自動化的封包監聽技術，精準提煉出 GET/POST 請求樣板，最終在背景高速驗證並產出可供後續爬蟲直接使用的完整分頁清單。

## 1. 需準備的輸入檔案格式

本系統具備三種執行模式，所需的輸入方式依模式而異：

* **[1] 執行大掃蕩 / [3] 單頁急先鋒**：
不需準備實體檔案。直接於命令提示字元 (Terminal) 中，貼上目標網站的起始網址（URL）即可（例如：`[https://example.com](https://example.com)`）。程式會自動補齊 `https://`。
* **[2] 審問嫌疑犯**：
系統會自動讀取同層目錄下的 **`suspect_targets.txt`** 純文字檔。此檔案由「模式 1」自動生成，格式為每一行一個帶有分頁特徵的嫌疑網址。

## 2. 預期的產出結果

程式執行完畢後，將會在腳本所在的同一層目錄下生成（或附加寫入）以下兩個純文字檔案：

1. **`suspect_targets.txt` (嫌疑名單)**：
在使用「[1] 大掃蕩」模式時，爬蟲會以廣度優先搜尋 (BFS) 往下深潛 3 層。只要「正則鷹眼」掃描到網頁內包含「第 X 頁」、「下一頁」或實體分頁按鈕，就會**即時寫入**此檔。不怕掃描中途中斷或當機。
2. **`final_ops_list.txt` (最終破解配置清單)**：
在使用「[2] 審問嫌疑犯」或「[3] 單頁急先鋒」成功捕捉到翻頁封包後，程式會在背景透過 `SessionPage` 無頭模式，自動替換參數並高速驗證後續頁面（最多向下挖掘 100 頁）。
* **GET 請求**：直接儲存完整的翻頁網址（如 `https://.../?page=2`）。
* **POST 請求**：儲存格式為 `目標網址#__POST__負載參數`，方便您後續快速拆解使用。
* *註：系統會透過 MD5 雜湊值比對每一頁的連結內容，若發現內容重複或無新連結，即判定到達「最後一頁」並自動停止寫入。*



## 3. 注意事項

在使用本工具前，請務必確認您的環境符合以下配置與操作規範：

* **必備第三方套件**：
執行前請開啟終端機並安裝強大的爬蟲控制套件 `DrissionPage`：
```bash
pip install DrissionPage

```


* **半自動化「封包監聽」操作流程**：
當進入「封包破解 Phase 2」時，程式會開啟一個可見的瀏覽器並啟動背景監聽。**此時需要您「手動」在該網頁上點擊第 2 頁、第 3 頁的按鈕**。待封包捕捉完成後，再回到終端機按下 `Enter` 鍵，程式才會開始進行參數差異比對（破解規律）。
* **防 OOM 記憶體保護機制**：
在執行大掃蕩時，為防止 Chromium 瀏覽器長時間開啟導致記憶體溢出（Out of Memory），系統設有 `RESTART_INTERVAL` 機制（預設每掃描 100 頁會自動重啟一次背景引擎），此為正常優化現象，請勿強行中斷程式。
* **高效佇列優化**：
V9.5 版已導入 `collections.deque` 處理掃描佇列，大幅改善了深層掃描時的記憶體操作瓶頸，確保大範圍巡邏時的運作流暢度。
